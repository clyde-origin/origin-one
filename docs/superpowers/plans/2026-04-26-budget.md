## Project rules quick reference (will bite you if you forget)

These are the ones most likely to cause a re-do; full text lives in root + back-to-one `CLAUDE.md`.

1. **Schema PRs are dedicated.** Never bundle a schema change with feature code. PRs 1, 3, 4 are pure schema; the rest assume the schema is already merged.
2. **`packages/schema` is sacred.** Schema-side TypeScript types in `packages/schema/src/*.ts` must be updated in lockstep with `packages/db/prisma/schema.prisma`. The Prisma client generates from `packages/db`; the hand-written types in `packages/schema` are what apps import.
3. **All three apps must compile after any schema change.** Run `pnpm -w build` (or at minimum `pnpm -w type-check`) on every schema PR before merging — `back-to-one`, `one-arc` (stub), `one-lore` (stub) all have to pass.
4. **`prisma generate` runs before any seed update.** The order is: schema PR → migrate → `pnpm --filter @origin-one/db exec prisma generate` → seed update if needed.
5. **Storage discipline.** New buckets ship with `auth.role() = 'authenticated'` RLS from day one and the bucket is created via Prisma migration, not a manual SQL script. Applies to PR 4 (`receipts` bucket).
6. **Replace-in-place navigation, no nested modals.** Sheet → tap → sheet replaces, not stacks (`apps/back-to-one/CLAUDE.md` → Design system).
7. **No optimistic updates.** React Query mutations always invalidate; matches existing convention in `queries.ts`.
8. **Reference HTML is the spec.** Match `apps/back-to-one/reference/*.html` exactly unless there's a documented reason not to.
9. **Verify branch before every commit.** `git status -sb` first — parallel CLI sessions can switch branches externally on shared worktrees (per saved memory).
10. **Feature branch per PR off `origin/main`.** This planning lives on `design/budget-explorations`; implementation never lands there. Every PR starts with `git fetch origin && git checkout -b feat/<name> origin/main`.

## Codebase patterns confirmed during execution (read before any UI/data PR)

Verified during PR 2 execution (#40) — these correct several assumptions in the original plan code blocks. **Subsequent PRs must follow these patterns**, not the per-PR code samples below where they diverge.

1. **Data layer is Supabase client, not Prisma.** Browser-side DB access in `apps/back-to-one/src/lib/db/queries.ts` uses `createBrowserAuthClient as createClient` from `@origin-one/auth`, then `db.from('TableName').select('*').eq(...)` etc. Inventory's helpers (`getInventoryItems`, `createInventoryItem`, `updateInventoryItem`, `deleteInventoryItem`) are the canonical template. The spec's `prisma.shootDay.findMany(...)` style appears in plan code blocks because I extrapolated wrong — substitute the Supabase shape when reading those blocks. **Server-side code (API routes in PR 13)** can use Prisma directly if needed, since those run on the server.

2. **React Query hooks live in `apps/back-to-one/src/lib/hooks/useOriginOne.ts`.** Pattern: register the query key in the top-of-file `keys` registry, then export `use<Thing>` (read), `useCreate<Thing>` / `useUpdate<Thing>` / `useDelete<Thing>` (mutations). Hooks invalidate via `qc.invalidateQueries({ queryKey: keys.thing(projectId) })`. Inventory hooks (line ~644) are the template.

3. **No project-level nav strip exists.** `apps/back-to-one/src/app/projects/[projectId]/layout.tsx` is `<HubContent> + <SubPageOverlay> + <ActionBar>`. **Discovery for new subpages = add a Hub preview block in `HubContent.tsx`**, sized like the Inventory block (header + horizontal chip strip with counts). The spec's §1 nav placement was corrected by Clyde in `0da492c` after PR #40 surfaced this.

4. **Hub blocks are rendered for everyone — gate them client-side via `readStoredViewerRole()`.** The viewer-identity shim lives at `apps/back-to-one/src/lib/utils/viewerIdentity.ts`. Pattern: `useState<ViewerRole | null>(null)` + `useEffect(() => setHubViewerRole(readStoredViewerRole()), [])` + conditionally render `{isProducer && (<block>)}`. Single swap point on Auth day; CrewPanel uses the same shim.

5. **ActionBar replaces the legacy FAB.** Per-page +-button behavior registers via `useFabAction({ onPress, label })` from `@/lib/contexts/FabActionContext`. `FabActionProvider` is mounted at the project layout. **Do NOT render a freestanding FAB**; the legacy bottom-right FAB component is gone (PR #37 finalized the migration). Some older mockups (e.g. `apps/back-to-one/reference/explorations/budget-page.html`) still depict a legacy FAB — those affordances need translating to ActionBar registration when their PR ships.

6. **Page-level producer redirect** (Q8). Subpages that should be producer-only (Schedule, Budget) are client components that read `readStoredViewerRole()` after hydration and `router.replace('/projects/<id>')` for non-producers. Render `null` pre-hydration to avoid a flash. PR 2's `app/projects/[projectId]/schedule/page.tsx` is the reference implementation.

7. **Types live in `apps/back-to-one/src/types/index.ts`** as plain TypeScript interfaces, not imported from `@origin-one/schema`. The Zod schemas in `packages/schema` are present but not directly consumed by app code today. New page-level types get added to the back-to-one types file mirroring the Prisma model shape (Decimal as `string`, Date as `string`, FKs nullable when nullable in schema).

```typescript
import type { ShootDay, ShootDayType } from '@origin-one/schema'

export async function getShootDays(projectId: string): Promise<ShootDay[]> {
  // mirror the pattern of getMilestones / getCrew — Prisma client via lib/db/prisma
  return prisma.shootDay.findMany({
    where: { projectId },
    orderBy: [{ date: 'asc' }, { sortOrder: 'asc' }],
  }) as unknown as ShootDay[]
}

export async function createShootDay(input: {
  projectId: string
  date: string
  type: ShootDayType
  notes?: string
  locationId?: string | null
}): Promise<ShootDay> {
  const maxOrder = await prisma.shootDay.aggregate({
    where: { projectId: input.projectId },
    _max: { sortOrder: true },
  })
  return prisma.shootDay.create({
    data: {
      projectId: input.projectId,
      date: new Date(input.date),
      type: input.type,
      notes: input.notes ?? null,
      locationId: input.locationId ?? null,
      sortOrder: (maxOrder._max.sortOrder ?? 0) + 1,
    },
  }) as unknown as ShootDay
}

export async function updateShootDay(id: string, patch: Partial<{
  date: string
  type: ShootDayType
  notes: string | null
  locationId: string | null
}>): Promise<ShootDay> {
  return prisma.shootDay.update({
    where: { id },
    data: {
      ...(patch.date !== undefined ? { date: new Date(patch.date) } : {}),
      ...(patch.type !== undefined ? { type: patch.type } : {}),
      ...(patch.notes !== undefined ? { notes: patch.notes } : {}),
      ...(patch.locationId !== undefined ? { locationId: patch.locationId } : {}),
    },
  }) as unknown as ShootDay
}

export async function deleteShootDay(id: string): Promise<void> {
  await prisma.shootDay.delete({ where: { id } })
}
```

import type { BudgetUnit, RateUnit } from '@origin-one/schema'

/**
 * Convert a CrewTimecard's (rateUnit, hours) into the (units, unit) pair
 * that becomes Expense.units / Expense.unit.
 *
 * Hour timecards: units = hours, unit = HOUR.
 * Day timecards:  units = 1,     unit = DAY  (one timecard date = one day —
 *                                              the hours field is informational
 *                                              only on day-unit timecards).
 *
 * This is the math PR #19 documented as broken; verified by tests in this PR.
 */
export function computeExpenseUnits(
  rateUnit: RateUnit,
  hours: number,
): { units: number; unit: BudgetUnit } {
  if (rateUnit === 'hour') return { units: hours, unit: 'HOUR' }
  return { units: 1, unit: 'DAY' }
}

/**
 * Resolve the rate to use when materializing a timecard into an Expense.
 *
 * Priority:
 *   1. timecard.rate            (the producer's per-timecard override)
 *   2. line.actualsRate         (per-line override on BudgetLine)
 *   3. rateSourceVersion's amount.rate
 *
 * Returns 0 when all three are null/undefined, so callers can multiply
 * safely without null guards. **Zero is treated as a valid explicit rate**
 * — an explicit 0 means "this work is unpaid", which is different from
 * "no rate set." Only null/undefined fall through.
 */
export function resolveExpenseRate(
  timecardRate: number | null | undefined,
  lineActualsRate: number | null | undefined,
  versionRate: number | null | undefined,
): number {
  if (timecardRate    !== null && timecardRate    !== undefined) return timecardRate
  if (lineActualsRate !== null && lineActualsRate !== undefined) return lineActualsRate
  if (versionRate     !== null && versionRate     !== undefined) return versionRate
  return 0
}

import { prisma } from '@/lib/db/prisma'

export async function cloneBudget(sourceProjectId: string, targetProjectId: string): Promise<string> {
  return prisma.$transaction(async (tx) => {
    const source = await tx.budget.findUniqueOrThrow({
      where: { projectId: sourceProjectId },
      include: {
        versions: true,
        accounts: { orderBy: { sortOrder: 'asc' } },     // parents before children — caller must verify
        lines:    { include: { amounts: true } },
        variables: true,
        markups:   true,
      },
    })

    const target = await tx.budget.create({
      data: {
        projectId: targetProjectId,
        currency: source.currency,
        varianceThreshold: source.varianceThreshold,
        clonedFromProjectId: sourceProjectId,
      },
    })

    // 1. Versions
    const versionIdMap = new Map<string, string>()
    for (const v of source.versions) {
      const created = await tx.budgetVersion.create({
        data: { budgetId: target.id, name: v.name, kind: v.kind, sortOrder: v.sortOrder, state: v.state },
      })
      versionIdMap.set(v.id, created.id)
    }
    if (source.rateSourceVersionId) {
      await tx.budget.update({ where: { id: target.id }, data: { rateSourceVersionId: versionIdMap.get(source.rateSourceVersionId)! } })
    }

    // 2. Accounts — depth-first, parents before children.
    const accountIdMap = new Map<string, string>()
    const sorted = topologicalAccounts(source.accounts)        // helper below
    for (const a of sorted) {
      const created = await tx.budgetAccount.create({
        data: {
          budgetId: target.id,
          parentId: a.parentId ? accountIdMap.get(a.parentId)! : null,
          section: a.section, code: a.code, name: a.name, sortOrder: a.sortOrder,
        },
      })
      accountIdMap.set(a.id, created.id)
    }

    // 3. Lines + amounts
    const lineIdMap = new Map<string, string>()
    for (const l of source.lines) {
      const created = await tx.budgetLine.create({
        data: {
          budgetId: target.id,
          accountId: accountIdMap.get(l.accountId)!,
          description: l.description, unit: l.unit, fringeRate: l.fringeRate,
          tags: l.tags, actualsRate: l.actualsRate, sortOrder: l.sortOrder,
        },
      })
      lineIdMap.set(l.id, created.id)
      for (const amt of l.amounts) {
        await tx.budgetLineAmount.create({
          data: {
            lineId: created.id,
            versionId: versionIdMap.get(amt.versionId)!,
            qty: amt.qty, rate: amt.rate, notes: amt.notes,
          },
        })
      }
    }

    // 4. Variables
    for (const v of source.variables) {
      await tx.budgetVariable.create({
        data: {
          budgetId: target.id,
          versionId: v.versionId ? versionIdMap.get(v.versionId)! : null,
          name: v.name, value: v.value, notes: v.notes,
        },
      })
    }

    // 5. Markups
    for (const m of source.markups) {
      await tx.budgetMarkup.create({
        data: {
          budgetId: target.id,
          versionId: m.versionId ? versionIdMap.get(m.versionId)! : null,
          accountId: m.accountId ? accountIdMap.get(m.accountId)! : null,
          name: m.name, percent: m.percent, appliesTo: m.appliesTo, sortOrder: m.sortOrder,
        },
      })
    }

    // Expenses are NOT cloned — actuals are real-world data per spec §7.2.
    return target.id
  })
}

function topologicalAccounts<T extends { id: string; parentId: string | null }>(rows: T[]): T[] {
  const sorted: T[] = []
  const remaining = [...rows]
  const placed = new Set<string>()
  while (remaining.length) {
    const i = remaining.findIndex(r => !r.parentId || placed.has(r.parentId))
    if (i < 0) throw new Error('cycle in account tree')
    const [next] = remaining.splice(i, 1)
    sorted.push(next)
    placed.add(next.id)
  }
  return sorted
}

import { evaluate, type EvalContext } from './eval'
import type { BudgetLine, BudgetLineAmount, BudgetMarkup, Expense } from '@origin-one/schema'

export interface ComputedLine {
  lineId: string
  qtyResolved: number
  qtyError: string | null
  rate: number
  fringeAmt: number
  total: number
  actuals: number
  varPct: number          // (actuals - total) / total
  flag: 'over' | 'warn' | 'under' | null
}

export function computeLine(
  line: BudgetLine,
  amount: BudgetLineAmount | undefined,
  expensesForLine: Expense[],
  ctx: EvalContext,
  varianceThreshold: number,
): ComputedLine {
  const qtyEval = amount ? evaluate(amount.qty, ctx) : { ok: true as const, value: 0 }
  const qty = qtyEval.ok ? qtyEval.value : 0
  const qtyError = qtyEval.ok ? null : describeEvalError(qtyEval.error)
  const rate = amount ? Number(amount.rate) : 0
  const fringeRate = Number(line.fringeRate)
  const fringeAmt = qty * rate * fringeRate
  const total = qty * rate + fringeAmt
  const actuals = expensesForLine.reduce((s, e) => s + Number(e.amount), 0)
  const varPct = total === 0 ? 0 : (actuals - total) / total
  const flag = total === 0 ? null
             : Math.abs(varPct) > varianceThreshold ? (varPct > 0 ? 'over' : 'under')
             : Math.abs(varPct) > varianceThreshold * 0.5 ? 'warn'
             : null
  return { lineId: line.id, qtyResolved: qty, qtyError, rate, fringeAmt, total, actuals, varPct, flag }
}

export async function syncExpenseFromTimecard(timecardId: string): Promise<void> {
  const t = await prisma.crewTimecard.findUniqueOrThrow({
    where: { id: timecardId },
    include: { crewMember: { include: { defaultLine: true } } },
  })

  const lineId = t.lineItemId ?? t.crewMember.defaultLineItemId
  if (!lineId) {
    // Producer-warning surface (toast / log) — not exception.
    console.warn(`[budget] timecard ${timecardId} not mapped to a line`)
    return
  }

  if (t.rateUnit == null) return            // backfill incomplete; skip & surface

  const line = await prisma.budgetLine.findUniqueOrThrow({ where: { id: lineId } })
  const budgetId = line.budgetId

  const { units, unit } = computeExpenseUnits(t.rateUnit, Number(t.hours))

  const unitRate = resolveExpenseRate(
    t.rate          != null ? Number(t.rate)          : null,
    line.actualsRate != null ? Number(line.actualsRate) : null,
    await fallbackToRateSource(budgetId, line.id),
  )
  const amount = units * unitRate

  await prisma.expense.upsert({
    where: { timecardId },
    create: {
      budgetId, lineId, source: 'timecard', amount: amount.toFixed(2),
      date: t.date, units: units.toFixed(2), unitRate: unitRate.toFixed(2), unit,
      timecardId, createdBy: t.approvedBy ?? t.crewMemberId,
    },
    update: {
      amount: amount.toFixed(2), units: units.toFixed(2), unitRate: unitRate.toFixed(2), unit, lineId,
    },
  })
}

  })
  return amt ? Number(amt.rate) : null

import type { BudgetLine, BudgetAccount, BudgetLineAmount, BudgetMarkup, Expense } from '@origin-one/schema'

export interface CsvInput {
  budgetId: string
  versions: { kind: string; name: string; id: string }[]
  accounts: BudgetAccount[]
  lines: (BudgetLine & { amounts: BudgetLineAmount[]; expenses: Expense[] })[]
  markups: BudgetMarkup[]
  rollups: { accountId: string; totalsByVersion: Record<string, number>; actuals: number }[]
  grandTotal: { totalsByVersion: Record<string, number>; actuals: number }
}

export function buildCsv(input: CsvInput): string {
  const rows: string[][] = []
  rows.push([
    'budget_id','version_kind','version_name',
    'account_code','account_name','account_path',
    'line_id','description','unit','qty_formula','qty_resolved',
    'rate','fringe_rate',
    'estimate_total','working_total','committed_total','actual_total',
    'variance_pct','tags','created_at','updated_at',
  ])
  // One row per line — version totals as columns. (Per spec §8.3 — NOT one row per line × version.)
  for (const line of input.lines) {
    rows.push([...lineRow(line, input)])
  }
  // Summary rows (per Q7) appended at end:
  for (const r of input.rollups) {
    const a = input.accounts.find(x => x.id === r.accountId)!
    rows.push(['','','', a.code, a.name, accountPath(a, input.accounts),
      '', `ACCOUNT TOTAL: ${a.name}`, '','','','','',
      String(r.totalsByVersion.estimate ?? 0), String(r.totalsByVersion.working ?? 0),
      String(r.totalsByVersion.committed ?? 0), String(r.actuals), '','','',''])
  }
  for (const m of input.markups) {
    rows.push(['','','', '','','', '', `MARKUP: ${m.name}`, '','','','',
      String(m.percent), '','','', '', '','','',''])
  }
  rows.push(['','','', '','','', '', 'GRAND TOTAL', '','','','','',
    String(input.grandTotal.totalsByVersion.estimate ?? 0),
    String(input.grandTotal.totalsByVersion.working ?? 0),
    String(input.grandTotal.totalsByVersion.committed ?? 0),
    String(input.grandTotal.actuals), '','','',''])

  return rows.map(r => r.map(csvEscape).join(',')).join('\n')
}

function csvEscape(s: string): string {
  if (s == null) return ''
  if (/[,"\n]/.test(s)) return `"${s.replace(/"/g, '""')}"`
  return s
}

// lineRow + accountPath omitted for brevity; ~20 lines each.
